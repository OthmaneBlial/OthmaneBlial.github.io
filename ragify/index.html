# rag_pipeline.py Streaming RAG Response 
async def generate_answer(query: str, app_id: UUID):
    kb = await knowledge_service.load(app_id)
    docs = retriever.hybrid(
        query=query,
        semantic_top_k=8,
        keyword_top_k=3,
    )
    context = summarizer.condense(docs)
    return router.stream(
        system_prompt="Grounded assistant with inline citations",
        messages=chat_history.for_app(app_id) + [
            {"role": "user", "content": query},
            {"role": "system", "content": context}
        ],
    )

Project Snapshot

  • Open source: MIT-licensed repository for teams exploring Retrieval-Augmented Generation patterns.
  • Backend: FastAPI + SQLAlchemy modules for knowledge bases, documents, chat, and model configs.
  • Frontend: Tailwind-powered demo UI showcasing chat, ingestion, and admin workflows.
  • Deploy anywhere: Dockerized stack with single-container startup scripts for quick pilots.

Key Technical Features

Modern infrastructure tuned for Retrieval-Augmented Generation

Every layer—from ingestion to inference—ships with measured defaults, predictable latency, and observability hooks for real deployments.

Preview it live at ragify-sqhu4om4aq-uc.a.run.app.

FastAPI Backend

Async streaming with typed contracts

Async endpoints, dependency overrides for testing, and automatic OpenAPI docs. Background tasks keep ingestion responsive while chats keep streaming.

Hybrid Database

SQLite/PostgreSQL with vectors

Switch between SQLite for local demos and PostgreSQL + pgvector in production. SQLAlchemy models keep migrations and connection pooling in sync.

Multi-model Router

100+ models via OpenRouter

Smart routing between GPT-4o, Claude, Llama, and more with per-application temperature, max_tokens, and safety configurations baked into ModelConfig.

Advanced Retrieval

Semantic + keyword fusion

BM25 for key phrases, transformer embeddings for semantic recall, and rerankers to ensure context blocks stay tightly scoped.

Production-ready

Single-container deployment

Docker Compose spins up backend, worker, and vector store. Health probes, metrics endpoints, and structured logging fit straight into your cluster.

Grounded Responses

Inline citations, scores, guardrails

Chat responses embed citation markers with document metadata and expose retrieval confidence for UI trust indicators.

Architecture Deep Dive

Layered system design

Each module is isolated so you can replace retrievers, swap vector stores, or plug in a new LLM without rewriting core flows.

1

Frontend UI

Vue-inspired component layout with chat viewport, file dropzone, and admin controls powered by vanilla JavaScript.

2

FastAPI Layer

Versioned REST endpoints with automatic validation, rate limiting hooks, and streaming Server-Sent Events for chat.

3

Application & Knowledge Modules

Dedicated services for applications, knowledge bases, and document processing keep domain logic isolated.

4

RAG Engine

Retriever, reranker, prompt composer, and generation router coordinate to build grounded answers with confidence scores.

5

Data Layer

SQLAlchemy models, Alembic migrations, and vector indexes (pgvector, LanceDB, Chroma) accessible through a single repository pattern.

6

External Integrations

OpenRouter for LLM access, Redis for caching, object storage for binary blobs, and observability hooks for tracing.

POST /api/v1/knowledge-bases/

Create a new knowledge base, attach metadata, and configure ingestion policies.

curl -X POST "http://localhost:8000/api/v1/knowledge-bases/" \
  -H "Content-Type: application/json" \
  -d '{
        "name": "Product Docs",
        "description": "Customer facing answers",
        "vector_config": {"dimension": 1536}
      }'
{
  "id": "kb_42",
  "name": "Product Docs",
  "document_count": 128,
  "status": "ready"
}

GET /api/v1/knowledge-bases/{id}

Fetch configuration, retention settings, and document statistics.

Pydantic Schemas

Typed contracts shared across services

Every API surface returns the same Pydantic models that power the frontend, ensuring zero drift between backend logic and UI expectations.

Application

Links assistants to knowledge bases, default prompts, and model presets.

class Application(BaseModel):
    id: UUID
    name: str
    knowledge_base_ids: List[UUID]
    model_config_id: UUID
    settings: Dict[str, Any]

KnowledgeBase

Organizes documents with lifecycle metadata and ingestion stats.

class KnowledgeBase(BaseModel):
    id: UUID
    name: str
    description: Optional[str]
    document_count: int
    status: Literal['ready', 'indexing']

Document

Captures ingestion source, chunking metadata, and processing logs.

class Document(BaseModel):
    id: UUID
    filename: str
    knowledge_base_id: UUID
    chunk_count: int
    metadata: Dict[str, Any]

ChatMessage

Stores user/bot pairs, citations, and latency for analytics.

class ChatMessage(BaseModel):
    role: Literal['user', 'assistant']
    content: str
    citations: List[Citation]
    created_at: datetime

ModelConfig

Defines provider, temperature, safety, and token budgets.

class ModelConfig(BaseModel):
    id: UUID
    provider: Literal['openrouter']
    model: str
    temperature: float
    max_tokens: int

SearchResult

Returns chunk text, citation handles, and relevance confidence.

class SearchResult(BaseModel):
    chunk: str
    source: str
    confidence: float
    metadata: Dict[str, Any]
